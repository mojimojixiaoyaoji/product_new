# 2. 需求背景与概述

## 2.1 需求背景

随着工业互联网的深度发展，企业工业数据量呈指数级增长，DCS（分布式控制系统）数据作为生产运营的核心数据，其查询、分析的效率直接影响企业生产监控、故障排查及战略决策的及时性。当前传统数据分析工具存在明显短板：需操作人员具备专业的SQL、数据分析技能，操作流程繁琐，数据查询与分析效率低下，无法满足企业各岗位（生产管理、技术、管理层、现场操作）对数据快速获取、深度分析的需求，亟需一款便捷、智能的专用查询工具解决上述痛点。

## 2.2 产品目标

基于AI大模型，打造一款适配工业场景的DCS数据查询助手，实现自然语言交互（支持手动输入、语音输入两种方式），快速响应工业数据集查询需求；针对企业核心查询、分析场景定制化设计，支持大模型精准识别用户查询意图，智能调用查询工具完成数据查询与多维度分析；具备智能会话记忆能力，支持多轮问答，降低操作门槛，提升各岗位数据获取与分析效率，为企业生产运营决策提供数据支撑。

## 2.4 产品亮点

1. **智能意图识别**：基于AI大模型（Qwen3-Next），精准识别复杂查询意图，可自动提取查询维度（总量、区域、工厂）、时间范围等核心信息，避免查询偏差。

2. **智能会话记忆**：依托AI大模型实现上下文深度理解，支持多轮问答，自动补全用户隐含意图，保持对话连贯性，无需重复输入相同查询条件。

3. **时间智能处理**：AI大模型自动识别“昨天、上周、上月”等相对时间，精准转换为具体日期范围，减少用户手动设置成本，提升查询准确性。

4. **多维度数据查询**：支持总量、区域、工厂三个核心维度，结合用户输入智能匹配查询维度，可实现单维度查询、多维度组合查询（如“北方区域各工厂采集量对比”）。

5. **图表可视化**：AI大模型自动分析数据，生成柱图、线图、饼图等可视化图表，直观展示数据趋势、占比等信息，无需手动制作图表。

6. **工厂和区域映射**：支持工厂简称、区域简称自动映射为标准全称（如“华蓥西南”映射为“四川华蓥西南水泥”），避免因简称输入导致的查询失败，提升查询准确性。

7. **自定义查询能力**：支持复杂查询场景，通过自然语言转SQL功能，实现灵活的自定义数据查询，满足专业人员深度分析需求。

8. **数据准确性保障**：遵循严格的数据处理规则，AI大模型对查询结果进行双重校验，确保数据处理的准确性、一致性，避免错误数据误导决策。

## 2.5 应用场景

1. **生产管理**：生产管理人员可实时查询各区域、工厂的DCS数据采集情况，快速识别数据异常（如采集量骤降、波动过大），及时介入处理，保障生产稳定。

2. **数据分析**：数据分析人员可快速获取历史数据趋势、多维度对比数据，无需手动整理，提升深度分析与预测效率，为生产优化提供数据支撑。

3. **决策支持**：管理层可通过区域对比、总量趋势等数据洞察，快速掌握全局生产情况，制定科学的生产计划、资源调配等战略决策。

4. **跨区域对比**：对比不同区域、工厂的生产数据，识别优势区域/工厂的管理经验，排查落后区域/工厂的问题，推动整体生产效率提升。

5. **日常监控**：相关岗位人员可定期（如每日、每周）监控数据采集情况，形成常态化监控机制，提前防范生产风险。

6. **问题排查**：生产出现异常时，技术人员可通过多轮问答快速查询相关数据（如异常时段、关联工厂数据），辅助故障定位与排查，缩短排查时间。

7. **绩效评估**：以数据采集情况为核心指标，评估各区域、工厂的生产绩效，为绩效考核、奖惩机制提供客观数据依据。

# 3. 产品概述

DCS数据查询助手是一款基于AI大模型（Qwen3-Next）开发的工业DCS数据专用查询与分析工具，聚焦企业各岗位核心数据需求，进行定制化设计。产品支持自然语言交互（涵盖手动输入与语音输入双模式），用户可通过简单提问，快速获取DCS数据查询结果、多维度趋势分析、区域/工厂对比等信息；具备智能会话记忆、多轮问答等核心功能，无需专业技术技能，全岗位可快速上手，旨在解决传统数据分析效率低、操作复杂的痛点，为企业生产运营、决策提供精准、高效的数据支撑。

# 4. 需求场景描述

本章节针对各核心用户角色，明确场景背景、核心痛点及产品解决方案，确保产品功能贴合实际业务需求，为后续用户故事、功能设计提供依据。

## 4.1 场景一：生产管理人员日常监控

- **背景**：生产管理人员需每日/每小时监控各区域、工厂的DCS数据采集情况，及时发现采集异常（如采集中断、数据骤降），确保生产系统正常运行。

- **痛点**：传统监控工具操作繁琐，需切换多个系统、手动筛选数据，无法快速获取核心监控信息，易错过异常处置最佳时机。

- **解决方案**：使用DCS数据查询助手，通过自然语言交互（支持手动输入、语音输入两种方式）或语音提问，快速获取核心监控数据及异常提示，无需复杂操作，提升监控效率。

## 4.2 场景二：技术人员故障排查

- **背景**：生产过程中出现设备故障、生产异常时，技术人员需分析DCS数据采集异常情况，定位故障原因，进行快速排查与修复。

- **痛点**：需手动登录多个数据系统，查询、整理相关数据，操作繁琐、效率低下，且无法快速关联多维度数据（如异常时段、关联工厂数据），延长故障排查时间。

- **解决方案**：使用DCS数据查询助手，通过自然语言交互（支持手动输入、语音输入两种方式）或语音提问获取详细的异常分析报告，支持多轮问答深入分析问题，无需手动整理数据、制作报表，缩短故障排查时间。

## 4.3 场景三：管理层决策支持

- **背景**：管理层需掌握全国各区域、工厂的DCS数据采集情况，了解生产整体态势，基于数据洞察制定生产计划、资源调配、区域管理等战略决策。

- **痛点**：缺乏直观的数据分析工具，需安排专人整理数据、制作报表，耗时较长，且无法快速获取多维度对比、趋势分析等核心信息，影响决策效率。

- **解决方案**：使用DCS数据查询助手，通过自然语言交互（支持手动输入、语音输入两种方式）或语音提问快速获取区域对比、总量趋势等数据，自动生成可视化图表，无需手动制作，提升决策效率。

## 4.4 场景四：现场操作人员快速查询

- **背景**：现场操作人员在生产现场作业时，需快速查询当前生产线、所在工厂的DCS数据，确认生产参数、采集情况，保障作业合规性。

- **痛点**：现场操作环境复杂（如双手需操作设备），不便使用键盘输入，传统查询方式无法满足现场快速查询需求。

- **解决方案**：使用DCS数据查询助手，通过自然语言交互（支持手动输入、语音输入两种方式）或语音提问快速查询当前生产线、所在工厂的DCS数据，确认生产参数、采集情况，适配现场操作场景，提升作业效率。

# 6. 用户故事 + GWT 定义

## 6.1 用户故事规范

- **格式要求**：作为 [角色]，我希望 [实现的功能/目标]，以便 [业务价值/用户收益]。

- **核心原则**：每个用户故事对应1-2个核心场景，配套GWT（Given-When-Then）定义，明确场景边界、前置条件、操作步骤及预期结果，为研发开发、测试验收提供清晰依据。

- **补充说明**：GWT定义需贴合实际业务场景，明确操作细节、响应时间、异常提示等关键信息，确保研发、测试可落地、可验证。

## 6.2 用户故事 + GWT 定义示例

### 6.2.1 生产管理人员

- **用户故事**：作为生产管理人员，我希望通过自然语言或语音快速查询工业DCS数据采集情况，以便及时发现生产异常并采取处置措施，保障生产稳定。

- **GWT定义-场景1：自然语言查询**

    - **Given**：用户已登录系统，角色为生产管理人员，拥有对应区域/工厂的数据查询权限；系统正常运行，已成功对接工业DCS数据源；用户明确查询需求（如具体工厂、时间范围）；用户可通过自然语言交互（支持手动输入、语音输入两种方式）提交查询。

- **GWT定义-场景2：语音输入查询**

    - **Given**：用户已登录系统，角色为生产管理人员，拥有对应区域/工厂的数据查询权限；系统语音识别功能正常，用户设备麦克风权限已开启；网络连接正常；用户可通过语音输入提交查询需求（与自然语言手动输入同属自然语言交互范畴）。

    - **When**：用户点击输入框左侧的语音按钮，系统显示“正在聆听...”提示，用户朗读「查询四川华蓥西南水泥今日DCS数据采集量」，朗读完成后点击确认识别（或系统自动停止识别）。

    - **Then**：语音识别准确率≥90%，正确转换为对应文本查询条件并填充至输入框；系统在3秒内返回该工厂今日采集总量、趋势图表及总结；若识别失败，系统返回明确提示：「语音识别失败，请重新朗读（建议语速适中、环境安静）」。

### 6.2.2 技术人员

- **用户故事**：作为技术人员，我希望通过多轮问答获取工业DCS数据采集异常的详细分析，以便快速定位故障原因，完成故障排查与修复。

- **GWT定义-场景：多轮问答异常分析**

    - **Given**：用户已登录系统，角色为技术人员，拥有数据分析、异常查询权限；系统已存储该区域/工厂的DCS数据采集历史；系统智能会话记忆功能已开启；网络连接正常。

    - **When**：用户在输入框中输入「近半年四川华蓥西南水泥DCS采集量波动原因分析」，点击发送按钮；系统返回波动原因分析结果后，用户继续输入「具体是什么原因导致1月份数据异常」。

    - **Then**：系统在5秒内完成分析并显示完整的波动原因分析报告（包含波动时段、波动范围、可能原因）；系统基于上一轮对话上下文，智能识别用户意图，针对1月份数据异常提供详细分析（如异常时段、关联参数、可能故障点）；支持查看异常时段的详细数据和趋势图表，辅助故障定位。

### 6.2.3 管理层

- **用户故事**：作为管理层，我希望通过区域对比了解各区域工业DCS数据采集情况，以便掌握全局生产态势，做出科学的战略决策。

- **GWT定义-场景：区域对比**

    - **Given**：用户已登录系统，角色为管理层，拥有全区域数据查看权限；系统已存储各区域工业DCS数据；系统智能会话记忆功能已开启；网络连接正常。

    - **When**：用户在输入框中输入「全国各区域今日DCS采集量对比」，点击发送按钮；系统返回各区域对比结果后，用户继续输入「北方区域为什么采集量较低」。

    - **Then**：系统在3秒内返回各区域今日采集量对比结果；结果包含各区域采集总量、占比图表（默认柱图，支持切换类型）、区域数据卡片；系统基于上一轮对话上下文，智能识别用户意图，针对北方区域采集量较低的原因提供详细分析（如工厂数量、生产负荷、设备状态等）；支持点击区域卡片查看该区域下属工厂的详细数据。

### 6.2.4 现场操作人员

# 7. 数据权限

为保障企业核心生产数据安全，系统采用精细化权限控制体系，权限分配基于用户角色，遵循“最小权限原则”，具体规则如下：

- **默认权限**：用户仅可见、可查询自身角色拥有调用权限的区域、工厂数据，无权限访问未授权数据（如管理层可查看全区域数据，现场操作人员仅可查看所在工厂/生产线数据）。

- **数据访问权限**：根据用户角色和权限级别，分级限制对敏感数据的访问，敏感数据（如异常故障详情、核心生产参数）仅授权给技术人员、管理层等相关岗位。

- **操作权限**：不同角色拥有不同的操作权限，具体如下：
        

    - 生产管理人员、现场操作人员：仅拥有数据查询、图表查看权限；

    - 技术人员：拥有数据查询、图表查看、异常分析、数据导出权限；

    - 管理层：拥有全权限（数据查询、图表查看、异常分析、数据导出、权限查看）。

- **权限管理**：系统支持管理员对用户角色、权限进行配置、修改、删除；记录权限变更日志，便于追溯。

## 9.2 业务流程

系统核心业务流程遵循“用户输入-意图识别-数据查询-结果展示-多轮交互”的逻辑，具体步骤如下：

1. 用户打开DCS数据查询助手页面，系统完成初始化（加载用户权限、历史查询记录、智能会话记忆、默认界面）；

2. 用户根据需求，选择对应功能模块（智能对话查询、数据总览、明细查询）；

3. 用户通过文本输入、语音输入或点击快捷按钮，提交查询需求；

4. 系统接收查询需求，若为语音输入，先通过Web Speech API转换为文本；

5. 系统通过Qwen3-Next大模型分析查询意图，提取核心参数（区域、工厂、时间范围），结合智能会话记忆处理上下文；

6. 系统根据查询意图，智能调用对应的数据接口（数据采集统计接口、趋势数据接口等）进行数据查询和分析；

7. 系统处理查询结果，生成数据卡片、表格、可视化图表（默认柱图），若有异常数据，添加异常提示；

8. 系统在右侧内容展示区呈现查询结果，用户可查看数据、切换图表类型、导出数据/图表；

9. 用户可基于当前结果继续提问，系统基于智能会话记忆，提供上下文相关的多轮问答服务；

10. 用户可在左侧历史查询记录中，查看、删除历史查询，或点击历史记录重新执行查询；

11. 用户退出系统后，系统保存用户历史查询记录、会话记忆（关联用户账号），下次登录可继续使用。

### 13.1 核心业务流程

#### 13.1.1 系统初始化流程
1. **页面加载**：用户打开DCS数据查询助手页面
2. **数据初始化**：系统初始化界面和默认数据，加载智能会话记忆
3. **界面渲染**：系统渲染左侧导航栏和右侧默认提示区域
4. **状态设置**：系统设置初始状态，准备接收用户输入

#### 13.1.2 查询处理流程
1. **接收输入**：系统接收用户的文本输入、语音输入或快捷按钮点击
2. **语音处理**：如果是语音输入，系统进行语音识别并转换为文本
3. **意图识别**：系统通过大模型分析查询意图，确定查询类型（总量查询、趋势分析、区域对比等）
4. **上下文处理**：系统结合智能会话记忆，处理上下文相关的查询
5. **工具调用**：系统智能调用查询工具进行数据查询和分析
6. **结果生成**：系统生成数据表格、图表等结果内容
7. **结果展示**：系统在右侧内容区展示查询结果
8. **会话记忆**：系统更新智能会话记忆，存储当前对话内容

### 13.2 功能点闭环

#### 13.2.1 智能对话闭环
- **输入**：用户输入自然语言或语音查询
- **处理**：系统通过大模型分析查询意图，生成查询结果
- **展示**：系统显示数据卡片、图表和表格
- **多轮问答**：用户可以基于当前结果继续提问，系统基于智能会话记忆提供上下文相关的回答
- **交互**：用户可以切换图表类型，形成闭环

#### 13.2.2 语音输入闭环
- **点击**：用户点击语音按钮
- **识别**：系统进行语音识别并转换为文本
- **确认**：用户确认识别结果
- **提交**：系统提交查询并返回结果
- **反馈**：系统提供语音识别状态的视觉反馈

#### 13.2.3 智能会话记忆闭环
- **初始化**：系统加载智能会话记忆
- **存储**：系统存储对话历史
- **调用**：系统在多轮问答中调用会话记忆
- **更新**：系统实时更新会话记忆
- **清理**：系统自动清理超出限制的会话记忆

#### 13.2.4 多轮问答闭环
- **第一轮**：用户提出初始问题
- **响应**：系统返回初始问题的答案
- **后续轮**：用户基于初始答案提出后续问题
- **上下文理解**：系统基于智能会话记忆理解上下文
- **相关回答**：系统返回与上下文相关的回答
- **循环**：重复上述过程，形成多轮问答闭环

#### 13.2.5 数据总览闭环
- **初始化**：系统加载行业云工业数据采集总量和趋势数据
- **展示**：系统生成默认柱图，显示近半年采集趋势和区域分布
- **切换**：用户可以切换图表类型（柱图、线图、饼图）
- **更新**：系统实时更新图表类型和样式

#### 13.2.6 明细查询闭环
- **筛选**：用户设置日期和工厂名称筛选条件
- **查询**：用户点击查询按钮提交请求
- **展示**：系统显示符合条件的工业数据采集明细列表
- **重置**：用户可以点击重置按钮清除筛选条件，形成闭环

#### 13.2.7 历史查询记录闭环
- **记录**：系统自动记录用户的查询历史
- **显示**：系统在左侧导航栏下方显示历史查询记录
- **点击**：用户可以点击历史查询记录重新执行查询
- **更新**：系统实时更新历史查询记录列表
